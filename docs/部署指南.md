# 看山小苗圃 · 部署指南

最后更新：2026-05-12（v0.6 线上部署补充）

---

## 1. 本地开发部署

### 1.1 前置条件

- Python 3.10+
- Node.js 20+
- PostgreSQL (可选，用于数据落盘)
- Redis (可选，用于缓存)

### 1.2 快速启动

```bash
# 1. 安装后端依赖
for s in services/*/; do pip install -r "$s/requirements.txt"; done
pip install -e packages/shared-python

# 2. 安装前端依赖
cd frontend && npm install && cd ..

# 3. 配置文件
cp services/config.example.yaml services/config.yaml
cp frontend/.env.example frontend/.env.local
# 编辑 .env.local，设置 NEXT_PUBLIC_KANSHAN_BACKEND_MODE=gateway

# 4. 一键启动
bash scripts/dev_up.sh
```

访问 http://127.0.0.1:3000

### 1.3 单独启动后端

```bash
bash scripts/run_all_services.sh
```

## 2. Docker 一键部署

### 2.1 前置条件

- Docker 20+
- Docker Compose v2+

### 2.2 部署步骤

```bash
# 1. 进入 infra 目录
cd infra

# 2. 复制环境变量
cp .env.deploy .env
# 按需编辑 .env 中的配置

# 3. 配置知乎凭证（可选）
cp services/config.example.yaml services/config.yaml
# 编辑 config.yaml 填入知乎凭证

# 4. 构建并启动
bash scripts/deploy.sh --build
```

### 2.3 常用命令

```bash
# 启动容器
bash scripts/deploy.sh

# 重新构建镜像
bash scripts/deploy.sh --build

# 查看日志
bash scripts/deploy.sh --logs

# 停止容器
bash scripts/deploy.sh --down
```

## 3. 线上服务器部署（单机 Docker Compose）

### 3.1 服务器要求

| 项目 | 最低 | 推荐 |
|------|------|------|
| 系统 | Ubuntu 22.04 / Debian 12 | Ubuntu 24.04 |
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 40 GB SSD | 80 GB SSD |
| Docker | 24+ | 最新稳定版 |
| Docker Compose | v2+ | 最新稳定版 |

需要一个域名，DNS A 记录指向服务器公网 IP。

### 3.2 部署步骤

#### Step 1: 服务器初始化

```bash
# 系统更新
sudo apt update && sudo apt upgrade -y

# 安装 Docker（Ubuntu）
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
# 重新登录使 docker 组生效

# 验证
docker --version
docker compose version

# 防火墙（只开必要端口）
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

#### Step 2: 拉取代码

```bash
sudo mkdir -p /opt/kanshan
sudo chown $USER:$USER /opt/kanshan
cd /opt/kanshan
git clone <your-repo-url> .
```

#### Step 3: 配置环境变量

```bash
cd /opt/kanshan/infra
cp .env.deploy .env
```

编辑 `.env`，**必须修改**以下项：

```bash
# 数据库密码（生产环境必须改）
POSTGRES_PASSWORD=<strong-random-password>
DATABASE_URL=postgresql+psycopg://kanshan:<strong-random-password>@postgres:5432/kanshan

# 存储后端
STORAGE_BACKEND=postgres
PROVIDER_MODE=live

# Redis
REDIS_URL=redis://redis:6379/0

# 知乎凭证（live 模式必填）
ZHIHU_APP_KEY=<your-app-key>
ZHIHU_APP_SECRET=<your-app-secret>
ZHIHU_ACCESS_TOKEN=<your-access-token>

# LLM 配置
LLM_PROVIDER=zhihu           # zhihu / openai_compat
LLM_API_BASE_URL=<api-url>   # 如使用 openai_compat
LLM_API_KEY=<api-key>        # 如使用 openai_compat

# 端口
FRONTEND_PORT=3000
API_GATEWAY_PORT=8000
```

#### Step 4: 配置知乎凭证

```bash
cd /opt/kanshan
cp services/config.example.yaml services/config.yaml
```

编辑 `services/config.yaml`：

```yaml
provider_mode: live
database_url: postgresql+psycopg://kanshan:<password>@postgres:5432/kanshan
storage_backend: postgres

zhihu:
  community:
    app_key: "<知乎社区 API user-token>"
    app_secret: "<app-secret>"
    base_url: "https://api.zhihu.com"
    writable_ring_ids:
      - "2029619126742656657"
    default_ring_id: "2029619126742656657"
  oauth:
    app_id: "<知乎开放平台 app_id>"
    app_key: "<知乎开放平台 app_key>"
    redirect_uri: "https://your-domain.com/api/v1/auth/zhihu/callback"
    base_url: "https://openapi.zhihu.com"
  data_platform:
    access_secret: "<知乎数据平台 access_secret>"
    base_url: "https://dip.zhihu.com"
    default_model: "zhida-thinking-1p5"
  quota:
    hot_list: 100
    zhihu_search: 1000
    global_search: 1000
    direct_answer: 100

cache:
  backend: redis
  redis_url: redis://redis:6379/0

logging:
  jsonl_dir: /app/output/logs
  console_level: INFO
```

#### Step 5: 配置 SSL + 反向代理

二选一：Nginx 或 Caddy。

**选项 A: Nginx**

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

创建 `/etc/nginx/sites-available/kanshan`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # Let's Encrypt 验证路径
    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL 证书（certbot 会自动填入）
    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # 安全头
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # gzip
    gzip on;
    gzip_types text/plain application/json application/javascript text/css;
    gzip_min_length 1000;

    # 请求体大小
    client_max_body_size 10m;

    # API 路由 → api-gateway
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }

    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }

    # 前端 → Next.js
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用并申请证书：

```bash
sudo ln -s /etc/nginx/sites-available/kanshan /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 申请 Let's Encrypt 证书
sudo certbot --nginx -d your-domain.com

# 验证自动续期
sudo certbot renew --dry-run
```

**选项 B: Caddy**

```bash
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy
```

创建或编辑 `/etc/caddy/Caddyfile`：

```
your-domain.com {
    # API 路由 → api-gateway
    handle /api/* {
        reverse_proxy localhost:8000
    }

    # 健康检查
    handle /health {
        reverse_proxy localhost:8000
    }

    # 前端 → Next.js
    handle {
        reverse_proxy localhost:3000
    }

    # 安全头
    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        X-Frame-Options "SAMEORIGIN"
        X-Content-Type-Options "nosniff"
        X-XSS-Protection "1; mode=block"
        Referrer-Policy "strict-origin-when-cross-origin"
    }

    # gzip
    encode gzip

    # 请求体大小
    request_body {
        max_size 10MB
    }
}
```

启动 Caddy（自动申请 HTTPS 证书）：

```bash
sudo systemctl enable caddy
sudo systemctl start caddy
```

#### Step 6: 启动服务

```bash
cd /opt/kanshan
bash scripts/deploy.sh --build
```

等待所有容器健康检查通过（约 1-2 分钟），然后验证：

```bash
# 检查容器状态
docker compose -f infra/docker-compose.yml ps

# 健康检查
curl -s https://your-domain.com/health | python3 -m json.tool
```

#### Step 7: 验证

浏览器访问 `https://your-domain.com`，确认：
- 前端页面正常加载
- 注册/登录功能正常
- 各 Tab 数据正常展示

### 3.3 生产环境配置要点

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| `PROVIDER_MODE` | mock | live |
| `STORAGE_BACKEND` | memory | postgres |
| `POSTGRES_PASSWORD` | dev_password | 强随机密码 |
| Redis | 无密码 | 设置 requirepass |
| SSL | 无 | Let's Encrypt / Caddy 自动 |
| 日志级别 | DEBUG | INFO |

### 3.4 数据备份

创建自动备份脚本 `/opt/kanshan/scripts/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/opt/kanshan/backups"
DATE=$(date +%F)
mkdir -p "$BACKUP_DIR"

# 备份 PostgreSQL
docker exec kanshan-postgres pg_dump -U kanshan kanshan | gzip > "$BACKUP_DIR/db-$DATE.sql.gz"

# 保留最近 30 天
find "$BACKUP_DIR" -name "db-*.sql.gz" -mtime +30 -delete

echo "Backup completed: $BACKUP_DIR/db-$DATE.sql.gz"
```

设置定时任务：

```bash
chmod +x /opt/kanshan/scripts/backup.sh
# 每天凌晨 3 点备份
echo "0 3 * * * /opt/kanshan/scripts/backup.sh >> /var/log/kanshan-backup.log 2>&1" | crontab -
```

恢复数据：

```bash
gunzip < /opt/kanshan/backups/db-YYYY-MM-DD.sql.gz | docker exec -i kanshan-postgres psql -U kanshan kanshan
```

## 6. 故障排查

| 症状 | 原因 | 处置 |
|------|------|------|
| 后端启动报 `kanshan_shared` 未找到 | shared-python 未安装 | `pip install -e packages/shared-python` |
| Gateway 503 SERVICE_NOT_READY | 下游服务未启动 | 检查各服务 `/health` |
| 前端白屏 | CORS 或 envelope 解包错误 | 检查浏览器 Console |
| PostgreSQL 连接失败 | 数据库未启动或密码错误 | 检查 `DATABASE_URL` |
| Redis 连接失败 | Redis 未启动 | 检查 `REDIS_URL` |
| JSONL 日志不生成 | 目录权限问题 | 检查 `output/logs/` 目录权限 |
