# 看山小苗圃 · 系统实现状态报告

最后更新：2026-05-12

---

## 1. 项目目的

看山小苗圃是一个面向知乎生态的**读写一体创作 Agent**，帮助用户完成完整的创作链路：

```
看到好内容 → 形成好观点 → 写出好文章
```

解决三个问题：
- **看什么**：基于用户画像，从热榜、搜索和关注流中筛选值得阅读的内容
- **想什么**：阅读后沉淀观点种子，通过疑问、反方、资料补强逐步发芽
- **写什么**：在观点成熟后辅助组织表达，经圆桌审稿产出有个人味的文章

核心原则：
- 不做一键水文生成
- 不绕过平台审核
- 不替用户伪造观点
- AI 输出默认是建议、草稿和辅助材料，最终表达由用户确认

```
│                                                             │

```

## 3. 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 前端 | Next.js 15 + TypeScript | 单页应用，7 个 Tab |
| 后端 | Python 3.10+ / FastAPI / Uvicorn | 9 个微服务 |
| 网关 | api-gateway | 纯反向代理，统一入口 |
| 数据库 | PostgreSQL 16 (pgvector) | 按 schema 隔离（profile/seed/writing/sprout/feedback） |
| 缓存 | Redis 7 | 知乎 API 缓存 + LLM 调用缓存 |
| 共享包 | packages/shared-python | 配置加载、结构化日志、数据库连接 |
| 容器 | Docker Compose | 12 个容器（9 服务 + 前端 + PG + Redis） |

---

## 4. 系统架构

```
前端 (Next.js :3000)
    │ HTTP
    ▼
API Gateway (:8000) ─── 代理 ──→ 8 个下游服务
    │
    ├── profile-service (:8010)   用户/认证/画像/Memory
    ├── content-service (:8020)   今日看什么/内容卡片
    ├── seed-service (:8030)      种子库/问题/材料
    ├── sprout-service (:8040)    今日发芽/写作机会
    ├── writing-service (:8050)   写作苗圃/8步状态机
    ├── feedback-service (:8060)  历史反馈/评论摘要
    ├── zhihu-adapter (:8070)     知乎API统一适配器
    └── llm-service (:8080)       LLM任务路由/多视角
```
