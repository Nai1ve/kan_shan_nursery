## 1. 本地开发部署

### 1.1 前置条件

- Python 3.10+
- Node.js 20+
- Docker（用于 PostgreSQL 和 Redis）

### 1.2 快速启动

```bash
# 1. 启动数据库和缓存
cd infra && docker compose up -d postgres redis && cd ..

# 2. 安装后端依赖
for s in services/*/; do pip install -r "$s/requirements.txt"; done
pip install -e packages/shared-python
pip install psycopg  # PostgreSQL 驱动

# 3. 安装前端依赖
cd frontend && npm install && cd ..

# 4. 配置文件
cp services/config.example.yaml services/config.yaml
cp frontend/.env.example frontend/.env.local

# 5. 启动后端服务
bash scripts/run_all_services.sh

# 6. 启动前端（新终端）
cd frontend && npm run dev
```

访问 http://localhost:3000

### 1.3 本地 dev 调试

如果要用 Docker 只承载 PostgreSQL / Redis，并在本机直接运行 9 个 Python 服务和前端 dev server，使用：

```bash
bash scripts/dev_up.sh
```

详细说明见 [本地非Docker开发调试.md](./本地非Docker开发调试.md)。

## 2. 线上服务器部署（单机 Docker Compose）

### 2.1 服务器要求

| 项目 | 最低 | 推荐 |
|------|------|------|
| 系统 | Ubuntu 22.04 / Debian 12 | Ubuntu 24.04 |
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 40 GB SSD | 80 GB SSD |
| Docker | 24+ | 最新稳定版 |
| Docker Compose | v2+ | 最新稳定版 |

### 2.2 部署步骤

#### Step 1: 服务器初始化

```bash
# 系统更新
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
# 重新登录使 docker 组生效

# 防火墙（只开必要端口）
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

#### Step 2: 拉取代码

```bash
sudo mkdir -p /opt/kanshan
sudo chown $USER:$USER /opt/kanshan
cd /opt/kanshan
git clone https://github.com/Nai1ve/kan_shan_nursery.git .
```

#### Step 3: 配置环境变量

```bash
cd /opt/kanshan/infra
cp .env.deploy .env
vi .env
```

**必须填写的值：**

```bash
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 前端访问后端的地址（浏览器端使用，必须公网可达）
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
KANSHAN_GATEWAY_URL=http://<你的服务器公网IP>:8000
KANSHAN_ZHIHU_URL=http://<你的服务器公网IP>:8070

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 数据库密码（必须改成强密码）
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POSTGRES_PASSWORD=<改成你的强密码>
DATABASE_URL=postgresql+psycopg://kanshan:<改成你的强密码>@postgres:5432/kanshan

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 存储后端（生产环境用 postgres）
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STORAGE_BACKEND=postgres

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# Provider 模式
# - mock: 使用模拟数据，不需要知乎凭证（测试用）
# - live: 使用真实知乎 API（需要知乎凭证）
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PROVIDER_MODE=mock
```

**如果使用 live 模式，还需要在 `services/config.yaml` 中填写知乎凭证（见 Step 4）。**

#### Step 4: 配置知乎凭证（live 模式必填）

```bash
cd /opt/kanshan
cp services/config.example.yaml services/config.yaml
vi services/config.yaml
```

**必须填写的值：**

```yaml
provider_mode: live
database_url: "postgresql+psycopg://kanshan:<你的密码>@postgres:5432/kanshan"
storage_backend: postgres

zhihu:
  community:
    # 来源：知乎个人主页 → "..." → 复制链接，取 people/ 后那串
    app_key: "<知乎 user-token>"
    # 来源：知乎开放平台申请
    app_secret: "<app-secret>"
    base_url: "https://api.zhihu.com"
    writable_ring_ids:
      - "<你的圈子 ID>"
    default_ring_id: "<你的默认圈子 ID>"

  oauth:
    # 来源：知乎开放平台 → 应用管理
    app_id: "<app_id>"
    app_key: "<app_key>"
    # 回调地址必须与知乎开放平台配置一致
    # 本地赛前调试使用前端回调页；上线前再改成正式同源域名
    redirect_uri: "http://127.0.0.1:3000/auth/zhihu/callback"
    base_url: "https://openapi.zhihu.com"

  data_platform:
    # 来源：https://developer.zhihu.com/ 个人中心
    access_secret: "<access_secret>"
    base_url: "https://developer.zhihu.com"
    default_model: "zhida-thinking-1p5"

  quota:
    hot_list: 100
    zhihu_search: 1000
    global_search: 1000
    direct_answer: 100

cache:
  backend: redis
  redis_url: "redis://redis:6379/0"

logging:
  jsonl_dir: /app/output/logs
  console_level: INFO
```

#### Step 5: 配置反向代理（可选但推荐）

**选项 A: Caddy（推荐，自动 HTTPS）**

```bash
sudo apt install -y caddy
```

创建 `/etc/caddy/Caddyfile`：

```
<你的域名> {
    handle /api/* {
        reverse_proxy localhost:8000
    }
    handle /health {
        reverse_proxy localhost:8000
    }
    handle {
        reverse_proxy localhost:3000
    }
    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        X-Frame-Options "SAMEORIGIN"
        X-Content-Type-Options "nosniff"
    }
    encode gzip
    request_body {
        max_size 10MB
    }
}
```

```bash
sudo systemctl enable caddy
sudo systemctl start caddy
```

**选项 B: Nginx**

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

创建 `/etc/nginx/sites-available/kanshan`：

```nginx
server {
    listen 80;
    server_name <你的域名>;

    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name <你的域名>;

    ssl_certificate     /etc/letsencrypt/live/<你的域名>/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/<你的域名>/privkey.pem;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    gzip on;
    gzip_types text/plain application/json application/javascript text/css;

    client_max_body_size 10m;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }

    location /health {
        proxy_pass http://127.0.0.1:8000;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/kanshan /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
sudo certbot --nginx -d <你的域名>
```

#### Step 6: 启动服务

```bash
cd /opt/kanshan
bash scripts/deploy.sh --build
```

等待所有容器健康检查通过（约 1-2 分钟）：

```bash
# 检查容器状态
docker compose -f infra/docker-compose.yml ps

# 健康检查
curl -s http://localhost:8000/health | python3 -m json.tool
```

#### Step 7: 验证

浏览器访问 `http://<你的IP>:3000` 或 `https://<你的域名>`，确认：
- 前端页面正常加载
- 注册/登录功能正常
- 各 Tab 数据正常展示

## 3.5 OAuth 上线前检查项

当前赛前调试使用 `127.0.0.1` 回调方案，避免 Cloudflare 临时域名失效和本地跨域问题。上线前必须重新处理：

```text
本地调试：
zhihu.oauth.redirect_uri = http://127.0.0.1:3000/auth/zhihu/callback

正式上线：
zhihu.oauth.redirect_uri = https://<domain>/auth/zhihu/callback
```

上线前检查：

- 知乎开放平台登记的 redirect URI 与 `services/config.yaml` 完全一致。
- 前端、OAuth 回调页、`/api/v1/auth/zhihu/callback` 对浏览器表现为同源或由同一公网域名代理。
- `api-gateway` 的 CORS 白名单包含正式前端域名。
- 如果没有稳定公网同源域名，则线上隐藏知乎 OAuth 入口，不让用户进入不可完成的授权流程。
- 修改 `zhihu.oauth.redirect_uri` 后重启 `zhihu-adapter`。

## 3.6 旧数据库结构热修

从旧版本数据库卷升级时，如果日志出现：

```text
sqlalchemy.exc.ProgrammingError: column users.setup_state does not exist
```

不要先删除数据库卷，执行增量 SQL：

```bash
docker exec -i kanshan-postgres psql -U kanshan -d kanshan <<'SQL'
ALTER TABLE profile.users
ADD COLUMN IF NOT EXISTS setup_state VARCHAR NOT NULL DEFAULT 'zhihu_pending';

UPDATE profile.users
SET setup_state = 'zhihu_pending'
WHERE setup_state IS NULL;
SQL
```

然后重启：

```bash
docker compose --env-file infra/.env -f infra/docker-compose.yml restart profile-service api-gateway
```

## 6. 故障排查

| 症状 | 原因 | 处置 |
|------|------|------|
| 前端报 `ERR_NAME_NOT_RESOLVED` | `KANSHAN_GATEWAY_URL` 配了 Docker 内部地址 | 改为公网 IP 或域名 |
| 注册/登录 401 | 用户不存在或密码错误 | 先注册，或检查密码 |
| PostgreSQL 连接失败 | 数据库未启动或密码错误 | 检查 `DATABASE_URL` 和 `POSTGRES_PASSWORD` |
| Redis 连接失败 | Redis 未启动 | `docker compose up -d redis` |
| 知乎 API 报错 | 凭证未配置或过期 | 检查 `services/config.yaml` |
| Docker Hub 拉取超时 | 网络问题 | 配置 Docker 镜像加速 |
