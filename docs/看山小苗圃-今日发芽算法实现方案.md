# 看山小苗圃-今日发芽算法实现方案

更新时间：2026-05-13

本文档为今日发芽算法的实现方案与变更记录，供后续校验使用。基于 `docs/看山小苗圃-高风险核心任务细化说明.md` 第 2 节规范实现。

---

## 一、实现概览

今日发芽算法从 mock 驱动的骨架升级为真实的服务端算法流水线：

```
用户手动触发 POST /sprout/start
→ 跨服务数据采集（种子、热点卡片、今日卡片、用户记忆）
→ 规则筛选候选种子
→ 确定性 6 因子激活评分
→ Top-K 候选送 LLM 生成机会
→ 持久化 run + opportunities
→ 返回结果
```

## 三、新增文件：filter.py

**路径：** `services/sprout-service/app/filter.py`

规则筛选纯函数模块，无外部依赖。

### 核心函数

```python
def filter_candidates(
    seeds: list[dict],
    dismissed_pairs: set[tuple[str, str]] | None = None,
    active_writing_seed_ids: set[str] | None = None,
) -> list[dict]:
```

### 纳入条件（全部满足）

1. `status not in ("published", "writing")`
2. `int(maturityScore) >= 45`
3. `len(wateringMaterials) >= 1`

### 排除条件（任一命中即排除）

1. 7 天内 dismissed 过的同 seed（通过 `dismissed_pairs` 检查）
2. 有 active writing session 的 seed（通过 `active_writing_seed_ids` 检查）
3. 空种子：`coreClaim` 和 `title` 都为空

### 辅助函数

```python
def build_dismissed_pairs_from_opportunities(
    opportunities: list[dict],
    days: int = 7,
) -> set[tuple[str, str]]:
```

扫描 dismissed opportunities，提取 7 天内的 `(seedId, triggerCardId)` 对。用于 in-memory 存储模式下构建 dismissed_pairs。

### 关键实现细节

- `maturityScore` 在数据库中是 String 类型，filter 内部做 `int()` 转换，异常时默认 0
- `dismissed_pairs` 来源于 `dismiss` 动作记录的 `(seedId, triggerCardId, dismissedAt)`
- 当 `triggerCardIds` 为空时，生成 `(seedId, "")` 作为种子级别的 dismissal 记录

## 四、新增文件：scorer.py

**路径：** `services/sprout-service/app/scorer.py`

激活评分纯函数模块，每个因子返回 `[0.0, 1.0]`，总分 clamp 到 `[0, 100]`。

### 权重常量

```python
WEIGHT_MATURITY = 0.25      # 种子成熟度
WEIGHT_RELATEDNESS = 0.25   # 话题相关度
WEIGHT_FRESHNESS = 0.20     # 时效性
WEIGHT_INFO_GAIN = 0.15     # 新信息增益
WEIGHT_MEMORY_FIT = 0.10    # 记忆匹配
WEIGHT_CONTROVERSY = 0.05   # 争议潜力
```

### 公式

```
activationScore = (
  seedMaturity * 0.25
+ topicRelatedness * 0.25
+ freshness * 0.20
+ newInfoGain * 0.15
+ memoryFit * 0.10
+ controversyPotential * 0.05
- penalties
) * 100
```

### 6 因子计算

| 因子 | 函数 | 计算方式 |
|------|------|----------|
| seedMaturity | `score_seed_maturity(seed)` | `maturityScore / 100`，clamp [0, 1] |
| topicRelatedness | `score_topic_relatedness(seed, hot, today)` | seed 文本与卡片标题的中文 bigram Jaccard 相似度，取所有卡片最大值；返回 `(score, bestTriggerCards, triggerType)` |
| freshness | `score_freshness(cards)` | 1天内=1.0, 3天内=0.9, 7天内=0.7, 更旧=max(0.3, 1-age/30), 无时间戳=0.7 |
| newInfoGain | `score_new_info_gain(seed, today_cards)` | 今日卡片能补充 seed 缺少的 evidence/counterargument/open_question 的比例；无需求时返回 0.3 |
| memoryFit | `score_memory_fit(seed, memory)` | interestId 命中 interestMemories=1.0, 标签部分匹配=0.7, 无匹配=0.3 |
| controversyPotential | `score_controversy_potential(seed, today_cards)` | `min(1.0, (len(counterArguments) + sum(controversies)) / 5)` |

### topicRelatedness P0 实现（bigram Jaccard）

```python
def _extract_bigrams(text: str) -> set[str]:
    """去标点空白，保留中文字符，生成 bigrams 集合（含单字）"""
    cleaned = re.sub(r"[^一-鿿㐀-䶿]", "", text)
    if len(cleaned) < 2:
        return set(cleaned) if cleaned else set()
    return {cleaned[i:i+2] for i in range(len(cleaned)-1)} | set(cleaned)

def _jaccard(a: set, b: set) -> float:
    """Jaccard = len(intersection) / len(union)"""
```

1. 对 seed 的 title + coreClaim + sourceTitle + material titles 拼接提取 bigrams
2. 对每张卡片的 title 提取 bigrams
3. 计算 Jaccard 相似度
4. 取最大值，同时记录 `bestTriggerCards`（最大值对应的卡片 ID 列表）和 `triggerType`（"hot" 或 "today_card"）

### 惩罚项

```python
def compute_penalties(seed, dismissed_pairs, published_topics) -> tuple[float, list[str]]:
```

| 条件 | 扣分 | reason 标识 |
|------|------|-------------|
| 证据材料为空 | -0.25 | `evidence_empty` |
| 只有摘要没有来源 | -0.15 | `no_source` |
| 最近 dismissed | -0.30 | `recently_dismissed` |
| 同类主题已发布 | -0.20 | `topic_already_published` |

### 主入口

```python
def compute_activation_score(
    seed, hot_cards, today_cards, memory,
    dismissed_pairs=None, published_topics=None,
) -> dict:
```

返回：
```json
{
  "total": 78.5,
  "factors": {
    "seedMaturity": 0.700,
    "topicRelatedness": 0.450,
    "freshness": 0.900,
    "newInfoGain": 0.300,
    "memoryFit": 1.000,
    "controversyPotential": 0.400
  },
  "penalties": -0.150,
  "penaltyReasons": ["no_source"],
  "bestTriggerCards": ["card-xxx"],
  "triggerType": "hot"
}
```

## 五、新增文件：data_fetcher.py

**路径：** `services/sprout-service/app/data_fetcher.py`

跨服务 HTTP 客户端，每个方法独立 try/except，失败返回空值 + WARNING 日志。

### 类定义

```python
class SproutDataFetcher:
    def __init__(self, service_urls):  # 接受 ServiceUrlsConfig
    def fetch_seeds(self, user_id: str) -> list[dict]: ...
    def fetch_hot_cards(self, limit: int = 10) -> list[dict]: ...
    def fetch_today_cards(self, user_id: str, limit: int = 10) -> list[dict]: ...
    def fetch_memory(self, session_id: str) -> dict: ...
    def fetch_all(self, user_id, session_id=None, hot_limit=10, today_limit=10) -> dict: ...
```

### 数据源映射

| 需求 | 服务 | 端点 | 认证 |
|------|------|------|------|
| 种子列表 | seed-service | `GET /seeds?user_id=X` | 无 |
| 热点卡片 | content-service | `GET /content/cards?limit=10` | 无（系统缓存） |
| 今日卡片 | content-service | `GET /content/cards?user_id=X&limit=10` | 无 |
| 用户记忆 | profile-service | `GET /memory/me` | x-session-id header |

### 并行采集

`fetch_all` 使用 `concurrent.futures.ThreadPoolExecutor(max_workers=4)` 并行采集，总超时 10 秒。返回 SproutInput 结构：

```json

  "seeds": [],
  "hotCards": [],
  "todayCards": [],
  "memory": {}
}
```

### 响应解析

- `_extract_items(data)`: 适配 gateway 信封格式 `{request_id, data: {items: [...]}}`
- `_extract_dict(data)`: 提取 `data` 字段或整个响应

## 六、修改文件：service.py

**路径：** `services/sprout-service/app/service.py`

从 mock 驱动重写为流水线编排。

### 构造函数变更

```python
# 旧
def __init__(self, storage=None, llm_client=None):

# 新
def __init__(self, storage=None, llm_client=None, data_fetcher: SproutDataFetcher | None = None):
```

新增 `data_fetcher` 参数和 `_dismissed_records` 列表。

### start_run 流水线

```python
def start_run(self, payload=None) -> dict:
    # Step 1: _gather_input(payload, user_id, session_id)
    #   → 优先用 payload 中的 seeds/hotCards/todayCards/memory
    #   → 缺失的通过 data_fetcher 补全
    #   → 按 interestId 过滤种子

    # Step 2: filter_candidates(seeds, dismissed_pairs)
    #   → 规则筛选

    # Step 3: compute_activation_score(seed, hot, today, memory, dismissed)
    #   → 对每个候选评分

    # Step 4: 按 total 降序排序，取 Top (limit+2)

    # Step 5: 调 LLM 或 fallback 构建机会
    #   → llm_client.sprout_opportunities(candidates, memory, limit)
    #   → 失败时 _build_fallback_opportunities()

    # Step 6: 持久化 run + opportunities
```

### P0 快捷模式

如果 payload 包含 `seeds`（且有 `hotCards` 或 `todayCards`），直接使用 payload 数据，不跨服务采集。两种模式可混合。

### 新增私有方法

| 方法 | 功能 |
|------|------|
| `_gather_input(payload, user_id, session_id)` | 数据采集编排 |
| `_call_llm(top_candidates, sprout_input, run_id, user_id, limit)` | LLM 调用 + 输出标准化 |
| `_build_fallback_opportunities(top_candidates, run_id, user_id, limit)` | LLM 失败时从候选直接构建机会 |
| `_get_dismissed_pairs(user_id)` | 从存储提取 7 天内 dismissed 的 (seedId, triggerCardId) 对 |
| `_record_dismissed(seed_id, trigger_card_ids)` | 记录 dismissed 对 |

### dismiss 动作变更

```python
# 旧：只设置 status
{**opportunity, "status": "dismissed"}

# 新：增加 dismissedAt + 记录 dismissed pair
{**opportunity, "status": "dismissed", "dismissedAt": _now_iso()}
self._record_dismissed(seed_id, trigger_card_ids)
```

### switch_angle 动作变更

优先调 LLM 换角度（如果 llm_client 可用），失败时用模板降级。

### LLM 输出标准化

`_call_llm` 为每个 LLM 返回的 opportunity 补充：
- `id`: `_create_id("sprout")`
- `runId`: 当前 run ID
- `userId`: 当前用户
- `score`: 用确定性评分（不用 LLM 评分），通过 seedId 匹配回 scored candidate
- `tags`: `[{"label": "发芽指数 XX", "tone": "blue"}]`
- `status`: `"active"`
- `createdAt`: ISO 时间戳

### 降级策略

```
完整流水线
→ 数据采集部分失败（用成功的部分）
→ 全部采集失败（用 payload 数据）
→ payload 也空且无种子（用 mock_data）
→ LLM 失败（从候选直接构建机会）
→ 0 候选有种子（返回空列表）
```

| 动作 | 端点 | 行为变更 |
|------|------|----------|
| 补充材料 | `POST /sprout/opportunities/{id}/supplement` | 不变 |
| 换角度 | `POST /sprout/opportunities/{id}/switch-angle` | 优先 LLM 重新生成，失败用模板 |
| 暂不写 | `POST /sprout/opportunities/{id}/dismiss` | 新增 dismissedAt 记录 + triggerCardIds 记录 |
| 开始写作 | `POST /sprout/opportunities/{id}/start-writing` | 不变 |
