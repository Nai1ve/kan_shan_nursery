# 看山小苗圃 · 前端接口文档

最后更新：2026-05-12

本文档是前端与后端的**接口契约**，记录所有 Gateway 端点、前端调用状态、请求/响应格式。

参考代码：
- `frontend/lib/types.ts` — TypeScript 类型定义
- `frontend/lib/api-client.ts` — 双模式读取客户端（mock / gateway）
- `frontend/lib/gateway-client.ts` — Gateway HTTP 客户端（信封解包）
- `frontend/lib/auth/auth-client.ts` — 认证客户端（注册/登录/OAuth）
- `services/api-gateway/app/main.py` — Gateway 路由（54 个端点）

---

## 1. 前端架构

### 1.1 双模式

通过 `NEXT_PUBLIC_KANSHAN_BACKEND_MODE` 环境变量切换：

| 值 | 行为 |
|---|------|
| `mock`（默认） | 读取来自 Next.js `/api/mock/*` 路由，写操作在 localStorage |
| `gateway` | 读取来自 api-gateway `:8000`，auth 走 gateway，写操作仍在 localStorage |

### 1.2 Gateway 响应格式

所有 Gateway 响应使用统一信封：

```json
// 成功
{ "request_id": "uuid", "data": { ... }, "meta": { ... } }

// 错误
{ "request_id": "uuid", "error": { "code": "ERROR_CODE", "message": "描述", "detail": { ... } } }
```

`gateway-client.ts` 自动解包，业务代码直接使用 `data`。

### 1.3 前端调用状态标记

本文档每个端点标注：
- **已接入** — 前端通过 api-client 或 auth-client 调用
- **未接入** — 后端已实现，前端未调用（写操作在 localStorage）

## 2. 认证 (Auth)

前端通过 `auth-client.ts` 调用，已接入 gateway 模式。

### 2.1 注册

```
POST /api/v1/auth/register
```

**前端状态**: 已接入（`auth-client.register()`）

请求：
```json
{ "nickname": "昵称", "email": "a@b.com", "username": "user1", "password": "****" }
```

响应：
```json
{ "user": { "userId": "...", "nickname": "...", ... }, "session": { "sessionId": "...", ... }, "setupState": "zhihu_pending" }
```

### 2.2 登录

```
POST /api/v1/auth/login
```

**前端状态**: 已接入（`auth-client.login()`）

请求：
```json
{ "identifier": "user1 或 a@b.com", "password": "****" }
```

响应：同注册。

### 2.3 登出

```
POST /api/v1/auth/logout
```

**前端状态**: 已接入（`auth-client.logout()`）

请求头：`x-session-id: {sessionId}`

### 2.4 查询当前用户

```
GET /api/v1/auth/me
```

**前端状态**: 已接入（`auth-client.getMe()`）

响应：
```json
{ "authenticated": true, "user": { "userId": "...", "nickname": "..." }, "setupState": "ready" }
```

### 2.5 知乎 OAuth

```
GET /api/v1/auth/zhihu/authorize       # 获取授权 URL
GET /api/v1/auth/zhihu/callback        # OAuth 回调（浏览器跳转）
GET /api/v1/auth/zhihu/binding         # 查询绑定状态
POST /api/v1/auth/zhihu/binding        # 创建绑定
DELETE /api/v1/auth/zhihu/binding      # 解除绑定
```

**前端状态**: authorize/binding(GET)/unbind 已接入，callback 由浏览器处理，binding(POST) 未接入。

### 2.6 Onboarding

```
POST /api/v1/profile/onboarding
```

**前端状态**: 已接入（`auth-client.saveOnboarding()`）

请求：
```json
{ "nickname": "...", "selectedInterests": [{ "interestId": "...", "selected": true, ... }], "writingStyle": { ... } }
```
