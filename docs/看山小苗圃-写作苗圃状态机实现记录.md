更新时间：2026-05-13

---

## 1. 实现目标

将 writing-service 从 6 状态扩展为规范要求的 9 状态完整链路，补齐蓝图确认、大纲生成/确认三个缺失阶段，重写圆桌为持久化状态机。

规范要求的链路：

```text
观点确认 → 蓝图 → 蓝图确认 → 大纲 → 大纲确认 → 草稿 → 圆桌讨论 → 定稿 → 发布
```

核心约束：不允许跳过蓝图直接生成正式草稿。

## 3. 状态机设计

### 3.1 九个状态

```text
claim_confirming     观点确认中（初始状态）
blueprint_ready      蓝图已生成，待确认
blueprint_confirmed  蓝图已确认
outline_ready        大纲已生成，待确认
outline_confirmed    大纲已确认
draft_ready          草稿已生成
reviewing            圆桌讨论中
finalized            已定稿
published            已发布（模拟）
```

### 3.2 状态转移表

`VALID_TRANSITIONS` 字典定义在 `session_logic.py`，作为唯一事实来源：

| 动作 | 起始状态 | 目标状态 |
|------|---------|---------|
| `create_session` | — | `claim_confirming` |
| `confirm_claim` | `claim_confirming` | `claim_confirming`（confirmed=True） |
| `generate_blueprint` | `claim_confirming` | `blueprint_ready`（需 confirmed） |
| `patch_blueprint` | `blueprint_ready` | `blueprint_ready` |
| `regenerate_blueprint` | `blueprint_ready` | `blueprint_ready` |
| `confirm_blueprint` | `blueprint_ready` | `blueprint_confirmed` |
| `generate_outline` | `blueprint_confirmed` | `outline_ready` |
| `patch_outline` | `outline_ready` | `outline_ready` |
| `regenerate_outline_section` | `outline_ready` | `outline_ready` |
| `confirm_outline` | `outline_ready` | `outline_confirmed` |
| `generate_draft` | `outline_confirmed` | `draft_ready` |
| `start_roundtable` | `draft_ready` | `reviewing` |
| `roundtable_author_message` | `reviewing` | `reviewing` |
| `continue_roundtable` | `reviewing` | `reviewing` |
| `adopt_suggestion` | `reviewing` | `reviewing` |
| `finalize` | `reviewing` / `draft_ready` | `finalized` |
| `publish_mock` | `finalized` | `published` |

守卫函数 `check_transition(current_status, action) -> target_status`，违反时抛 `InvalidTransition`。

### 3.3 跳过防护

- `generate_blueprint` 额外检查 `session["confirmed"]`，未确认不能生成蓝图
- `generate_draft` 仅允许从 `outline_confirmed` 进入，不能跳过蓝图或大纲
- `publish_mock` 仅允许从 `finalized` 进入

## 4. 数据结构

### 4.1 WritingBlueprint（规范对齐）

```python
{
    "centralClaim": str,          # 核心观点
    "mainThread": str,            # 主线论述
    "argumentSteps": [            # 论证步骤
        {
            "id": "arg-xxx",
            "title": str,
            "purpose": str,
            "keyPoints": [str]
        }
    ],
    "counterArguments": [str],    # 反方论点
    "responseStrategy": str,      # 回应策略
    "personalExperienceNeeded": [str],  # 需要的个人经验
    "riskNotes": [str]            # 风险提示
}
```

旧结构 `{background, arguments, counterArgument, suggestedPersonalExperience}` 已废弃。

### 4.2 WritingOutline（新增）

```python
{
    "sections": [
        {
            "id": "sec-xxx",
            "title": str,
            "purpose": str,
            "keyPoints": [str],
            "referencedMaterialIds": [str],   # 引用的材料 ID
            "referencedSourceIds": [str],     # 引用的来源 ID
            "missingMaterialHints": [str]     # 缺失材料提示
        }
    ]
}
```

每个 `blueprint.argumentSteps` 映射为一个 outline section。

### 4.3 RoundtableState（持久化）

```python
{
    "status": "active",           # not_started | active | closed
    "turns": [                    # 讨论轮次
        {
            "id": "turn-xxx",
            "role": "author" | "logic_reviewer" | "human_editor" | "opponent_reader" | "community_editor",
            "content": str,
            "createdAt": str
        }
    ],
    "suggestions": [              # 可采纳的建议
        {
            "id": "sug-xxx",
            "fromRole": str,
            "content": str,
            "severity": "high" | "medium" | "low",
            "adopted": bool
        }
    ]
}
```

旧实现圆桌结果不持久化，现在完整存储在 session 中。

### 4.4 WritingSession 完整字段

```python
{
    "sessionId": "ws-xxx",
    "seedId": str,
    "interestId": str,
    "articleType": str,           # 默认 "工程复盘"
    "coreClaim": str,
    "memoryOverride": dict,       # 兴趣记忆覆盖
    "tone": "balanced" | "sharp" | "steady",
    "confirmed": bool,            # 观点是否已确认
    "adoptedSuggestions": [str],  # 已采纳建议 ID（兼容旧字段）
    "draftStatus": str,           # 九个状态之一
    "savedDraft": bool,
    "blueprint": dict,            # WritingBlueprint（blueprint_ready 后有值）
    "outline": dict,              # WritingOutline（outline_ready 后有值）
    "draft": dict,                # WritingDraft（draft_ready 后有值）
    "roundtable": dict,           # RoundtableState（reviewing 后有值）
    "publishedArticleId": str     # published 后有值
}
```

## 6. LLM 集成

### 6.1 涉及的 LLM 任务类型

| 任务类型 | 模式 | Prompt 文件 | 用途 |
|---------|------|------------|------|
| `argument-blueprint` | single | `prompts/v1/argument-blueprint.md` | 生成蓝图 |
| `generate-outline` | single | `prompts/v1/generate-outline.md` | 生成大纲（新增） |
| `draft` | single | `prompts/v1/draft.md` | 生成草稿 |
| `roundtable-review` | multi_persona | `prompts/v1/roundtable-review.md` + 4 个 persona | 圆桌讨论 |

### 6.2 generate-outline 任务

**输入：**

```json
{
    "taskType": "generate-outline",
    "input": {
        "blueprint": { "centralClaim", "argumentSteps", ... },
        "materials": [],
        "memory": {}
    }
}
```

**输出：**

```json
{
    "sections": [
        {
            "id": "section-xxx",
            "title": "章节标题",
            "purpose": "本节作用",
            "keyPoints": ["要点"],
            "referencedMaterialIds": [],
            "referencedSourceIds": [],
            "missingMaterialHints": ["建议补充"]
        }
    ]
}
```

### 6.3 LLM Client 方法

`WritingLlmClient`（`writing-service/app/llm_client.py`）四个方法：

| 方法 | 调用任务 |
|------|---------|
| `argument_blueprint(seed, materials, memory, article_type)` | `argument-blueprint` |
| `generate_outline(blueprint, materials, memory)` | `generate-outline`（新增） |
| `draft(seed, materials, blueprint, memory, tone)` | `draft` |
| `roundtable_review(seed, draft, memory)` | `roundtable-review` |

所有方法均遵循 try-catch + mock fallback 模式。

### 6.4 Builder 函数（session_logic.py）

| 函数 | LLM 调用 | fallback |
|------|---------|----------|
| `_build_blueprint()` | `llm_client.argument_blueprint()` | 硬编码蓝图结构 |
| `_build_outline()` | `llm_client.generate_outline()` | 从 blueprint.argumentSteps 派生 |
| `_build_draft()` | `llm_client.draft()` | 硬编码草稿 |
| `_init_roundtable_state()` | `llm_client.roundtable_review()` | 4 个硬编码 persona 意见 |
| `_continue_roundtable()` | `llm_client.roundtable_review()` | 追加一轮 logic_reviewer 回复 |

## 9. 已知限制与后续事项

1. **前端未对接**：前端当前在本地 React state 管理写作流程，未调用 gateway writing 接口。需要前端改造对接。
2. **大纲 section 的 referencedMaterialIds 为空**：mock 实现中不填入真实材料 ID，需接入 seed-service 的材料数据后才能真正映射。
3. **圆桌 continue_roundtable 的 LLM prompt**：当前 roundtable-review prompt 未显式支持 turns 历史输入，mock 直接追加一轮回复。后续需更新 prompt 支持多轮上下文。
4. **旧 session 兼容**：已存在的 session 使用旧蓝图结构 `{background, arguments, counterArgument}`，新代码读取 `argumentSteps` 会为空。建议清理旧数据或重写迁移逻辑。
5. **draft.md prompt 结构对齐**：`draft` prompt 期望 `blueprint` 输入，当前传入新结构（含 `argumentSteps`）。mock 路径无影响，真实 LLM 路径需验证 prompt 是否能正确解析。
