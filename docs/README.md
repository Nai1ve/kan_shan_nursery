# docs 目录说明

更新时间：2026-05-14

本文档是 `docs` 目录的索引。根目录 [README.md](../README.md) 负责项目总览，本文件负责把产品、算法、接口、部署和协作文档归类，避免开发时继续引用过时设计。

## 合并口径

本轮已经把重复文档收敛为“主文档 + 归档文档”：

| 主题 | 当前主文档 | 已归档内容 |
|:---|:---|:---|
| 产品说明与路演表达 | [看山小苗圃-产品文档.md](看山小苗圃-产品文档.md)、[看山小苗圃-项目完成度与Demo说明.md](看山小苗圃-项目完成度与Demo说明.md) | 旧版产品说明、polished 长文、阶段性进度状态 |
| 核心流程与算法 | [看山小苗圃-核心流程校准方案.md](看山小苗圃-核心流程校准方案.md)、今日看什么/今日发芽/历史反馈等专题文档 | 旧 HTML 原型、旧算法总文档、旧 Agent 体系 |
| 部署与 OAuth | [看山小苗圃-实际部署清单与一键脚本.md](看山小苗圃-实际部署清单与一键脚本.md)、[部署指南.md](部署指南.md)、[本地非Docker开发调试.md](本地非Docker开发调试.md) | 单独 IP OAuth 快速方案、Ubuntu 长部署方案、旧 OAuth 跑通方案 |

新任务默认只引用当前主文档；归档文档只用于回看历史思路。

## 推荐阅读顺序

给 AI Coding 或人工开发分任务时，建议按下面顺序阅读：

```text
1. README.md
2. docs/看山小苗圃-产品文档.md
3. docs/看山小苗圃-核心流程校准方案.md
4. docs/看山小苗圃-接口功能与数据流文档.md
5. docs/看山小苗圃-服务拆分与开发协作.md
6. 被改服务的 README.md
7. packages/shared-schemas 下对应 schema
```

归档原则：

- 旧 HTML 原型不再作为前端实现依据。

- 旧单点部署/OAuth 文档已由实际部署清单、本地 Dev 调试和部署指南覆盖。

- 如需回看历史思路，可以阅读 archive；新开发任务不要引用 archive 中的文档作为验收标准。
