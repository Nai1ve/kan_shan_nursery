# 看山小苗圃 - 数据库结构定稿

> 最后更新：2026-05-13
> 版本：v1.0.0

## 概述

看山小苗圃使用 PostgreSQL 数据库，按服务划分为 5 个 schema：

| Schema | 所属服务 | 用途 |
|--------|----------|------|
| `profile` | profile-service | 用户、会话、绑定、画像 |
| `seed` | seed-service | 种子内容管理 |
| `writing` | writing-service | 写作会话管理 |
| `sprout` | sprout-service | 发芽机会与运行记录 |
| `feedback` | feedback-service | 反馈文章管理 |

---

## 1. profile Schema

### 1.1 users - 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| user_id | VARCHAR | PRIMARY KEY | 用户ID，格式: `usr-{uuid12}` |
| nickname | VARCHAR | NOT NULL | 昵称 |
| email | VARCHAR | UNIQUE | 邮箱（可选） |
| username | VARCHAR | UNIQUE | 用户名（可选） |
| password_hash | VARCHAR | NOT NULL | bcrypt 密码哈希 |
| created_at | VARCHAR | NOT NULL | ISO 8601 创建时间 |
| setup_state | VARCHAR | NOT NULL, DEFAULT 'zhihu_pending' | 用户设置状态 |

**setup_state 枚举值：**
- `zhihu_pending` - 等待绑定知乎账号
- `llm_pending` - 等待配置 LLM
- `ready` - 配置完成，可正常使用

### 1.2 sessions - 会话表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| session_id | VARCHAR | PRIMARY KEY | 会话ID |
| user_id | VARCHAR | NOT NULL | 关联用户ID |
| created_at | VARCHAR | NOT NULL | ISO 8601 创建时间 |
| expires_at | VARCHAR | | ISO 8601 过期时间 |

### 1.3 zhihu_bindings - 知乎绑定表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| user_id | VARCHAR | PRIMARY KEY | 关联用户ID |
| zhihu_uid | VARCHAR | | 知乎用户UID |
| access_token | TEXT | | 知乎访问令牌 |
| binding_status | VARCHAR | NOT NULL, DEFAULT 'not_started' | 绑定状态 |
| bound_at | VARCHAR | | ISO 8601 绑定时间 |
| expired_at | VARCHAR | | ISO 8601 过期时间 |

**binding_status 枚举值：**
- `not_started` - 未开始
- `bound` - 已绑定
- `expired` - 已过期

### 1.4 profile_data - 画像数据表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR | PRIMARY KEY, DEFAULT 'default' | 记录ID |
| data | TEXT | NOT NULL | JSON 格式画像数据 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

### 1.5 profile_versions - 画像版本表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR | PRIMARY KEY | 版本ID |
| target | VARCHAR | NOT NULL | 版本目标 |
| snapshot | TEXT | NOT NULL | JSON 格式快照 |
| reason | VARCHAR | NOT NULL | 变更原因 |
| created_at | VARCHAR | NOT NULL | ISO 8601 创建时间 |

### 1.6 memory_update_requests - 记忆更新请求表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR | PRIMARY KEY | 请求ID |
| interest_id | VARCHAR | NOT NULL | 兴趣ID |
| target_field | VARCHAR | NOT NULL | 目标字段 |
| suggested_value | TEXT | NOT NULL | 建议值 |
| reason | TEXT | NOT NULL | 更新原因 |
| status | VARCHAR | NOT NULL, DEFAULT 'pending' | 状态 |
| created_at | VARCHAR | NOT NULL | ISO 8601 创建时间 |

**status 枚举值：**
- `pending` - 待处理
- `approved` - 已批准
- `rejected` - 已拒绝

### 1.7 writing_styles - 写作风格表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| user_id | VARCHAR | PRIMARY KEY | 关联用户ID |
| logic_depth | INTEGER | DEFAULT 3 | 逻辑深度 (1-5) |
| stance_sharpness | INTEGER | DEFAULT 3 | 立场锐度 (1-5) |
| personal_experience | INTEGER | DEFAULT 3 | 个人经历 (1-5) |
| expression_sharpness | INTEGER | DEFAULT 3 | 表达锐度 (1-5) |
| uncertainty_tolerance | INTEGER | DEFAULT 3 | 不确定性容忍 (1-5) |
| preferred_format | VARCHAR | DEFAULT 'long_article' | 偏好格式 |
| evidence_vs_judgment | VARCHAR | DEFAULT 'balanced' | 证据vs判断 |
| opening_style | VARCHAR | DEFAULT 'question' | 开头风格 |
| title_style | VARCHAR | DEFAULT 'controversy' | 标题风格 |
| emotional_temperature | VARCHAR | DEFAULT 'rational' | 情感温度 |
| ai_assistance_boundary | VARCHAR | DEFAULT 'draft_only' | AI辅助边界 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

**preferred_format 枚举值：**
- `long_article` - 长文
- `short_article` - 短文
- `thread` - 连载

**evidence_vs_judgment 枚举值：**
- `evidence_heavy` - 重证据
- `balanced` - 平衡
- `judgment_heavy` - 重判断

**opening_style 枚举值：**
- `question` - 提问式
- `story` - 故事式
- `data` - 数据式

**title_style 枚举值：**
- `controversy` - 争议型
- `curiosity` - 好奇型
- `direct` - 直接型

**emotional_temperature 枚举值：**
- `rational` - 理性
- `warm` - 温暖
- `passionate` - 激情

**ai_assistance_boundary 枚举值：**
- `draft_only` - 仅草稿
- `outline_only` - 仅大纲
- `full_assist` - 全程辅助

### 1.8 llm_configs - LLM配置表

| 字段 | 类型 | 约束 | 说明 |

| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

## 部署说明

### 初始化数据库

```bash
# 使用 init.sql 初始化
psql -U postgres -d kanshan -f infra/postgres/init.sql
```

### 迁移注意事项

从旧版本升级时，需要执行以下 SQL：

```sql
-- 添加 setup_state 字段到 users 表
ALTER TABLE profile.users ADD COLUMN IF NOT EXISTS setup_state VARCHAR NOT NULL DEFAULT 'zhihu_pending';
```

### 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| DATABASE_URL | PostgreSQL 连接字符串 | - |
| OAUTH_STATE_SECRET | OAuth state 签名密钥 | kanshan-oauth-state-dev-secret |
